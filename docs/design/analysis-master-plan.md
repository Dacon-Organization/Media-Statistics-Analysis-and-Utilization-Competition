|---|--------|----------------|------|
| 1 | 데이터 내용 파악 부족 | 인코딩 3종 혼재·가중치 변수명 4종·2022 구조 이상(N=58,936)을 **이번에야 발견** | **Data Understanding 단계 부재** → 신설 |
| 2 | 2025 단일연도만 진행 | 7개년 보유했으나 1개년만 사용. 게다가 **변수명 직결 불가**(공통 교집합 2개) | **7개년 하모나이즈 전략 필요** |
| 3 | 실무 워크플로(전처리·정제·마이닝·모델링) 부재 | 정제 파이프라인 문서 없음, 모델은 K-means 단일, 모델 타당도 정식 검증 없음 | **CRISP-DM 6단계 채택** |

## 1. 데이터 자산 실측 프로파일 (2026-06-24 메타 스캔)

7개년 `.sav` 메타데이터를 `pyreadstat`(metadataonly)로 직접 스캔한 결과. **이 표가 Data Understanding의 출발점.**

| 연도 | 표본 N | 변수 수 | 인코딩 | 가중치 변수 | 비고 |
|:--:|:--:|:--:|:--:|:--:|------|
| 2019 | 5,040 | 286 | euc-kr | `wt1`·`wt2` | 복수 가중치 |
| 2020 | 5,010 | 281 | euc-kr | `WT` | |
| 2021 | 5,010 | 307 | **utf-8** | `WT` | 인코딩 분기점 |
| 2022 | **58,936** | 244 | **utf-8** | `WT` | ⚠️ 개인용(16.7MB)인데 표본 과다 → **long-format/가구원 전수 의심, 구조 재확인 필수** |
| 2023 | 5,000 | 245 | euc-kr | `HMWT` | 가중치명 상이 |
| 2024 | 6,000 | 237 | euc-kr | `WT` | |
| 2025 | 6,000 | 223 | euc-kr | `WT` | 분석 메인 |

### 1.1 발견된 정합성 난제 (실측)
- **인코딩 혼재**: euc-kr(19·20·23·24·25) vs utf-8(21·22). 단일 인코딩 로더는 2021에서 크래시함(실측).
- **가중치 변수명 4종**: `wt1/wt2`(2019)·`WT`(20·21·22·24·25)·`HMWT`(2023). → 통일 매핑 필요.
- **변수명 직결 불가**: 7개년 공통 변수명 교집합 = **단 2개**. 인접연도 Jaccard도 들쭉날쭉(20-21: 0.73 / 21-22: 0.10 / 22-23: 0.12 / 23-24: 0.71). → **문항번호 체계가 2~3년마다 리셋**. 변수명이 아니라 **문항 의미 기반 crosswalk(매핑 테이블)** 를 코드북으로 수작업 구축해야 추세분석 가능.
- **핵심 신뢰문항 위치 이동**: `Q84·Q95·Q96`이 2025에만 존재(타 연도는 번호 다름).
- **2022 표본 이상**: N=58,936. 분석 단위(개인 vs 가구원) 재정의 후 정제 필요.

> 이 난제들이 곧 **Data Preparation의 핵심 작업량**이다. (지금까지 "2025만" 봤기에 안 보였던 부분)

## 3. Phase별 상세 설계

### P2 — Data Understanding (다음 세션 1순위)
1. **7개년 통합 로더**: 인코딩 자동탐지(euc-kr→utf-8 fallback) + 가중치 변수명 통일(`wt1/WT/HMWT`→표준 `WT`). `src/load_audience.py`(신규).
2. **변수 카탈로그**: 연도×변수 존재행렬, 레이블 텍스트 수집 → `docs/design/variable-crosswalk.md` 초안.
3. **품질 프로파일**: 연도별 결측율·특수코드 분포·이상치·표본설계(2022 구조 규명).
4. **Crosswalk 매핑(수작업+반자동)**: 문항 레이블 텍스트 유사도(가벼운 문자열/임베딩)로 후보 매칭 → 코드북 PDF로 사람이 확정. 대상 = 신뢰·매체이용·인구 핵심지표 우선.

### P3 — Data Preparation
1. **정제 규칙 SSOT화**: 특수코드→결측, 역코딩, 가중치 정규화, 연령 비닝(기존 규칙 흡수).
2. **결측 처리 전략**: 구조적 결측(게이트 문항) vs 무응답 구분 → 대치(MICE/단순) 또는 모델 내 처리. 근거는 Perplexity 융합(§6).
3. **이상치·일관성**: 일수>7, 논리 모순 응답 검출.
4. **하모나이즈 데이터셋 산출**: `data/interim/harmonized_panel.parquet`(연도 공통 핵심지표 long-format). 반복횡단면(repeated cross-section) 구조.
5. **파생변수**: 신뢰·다양성 지수의 다개년 버전, 코호트(출생연도=조사연도−연령) 키.

### P4 — Modeling (§5 기법 위계 참조)
- 횡단면(2025 메인): 요인분석→군집→분류→회귀/인과.
- 종단(7개년): 추세검정·연령-기간-코호트(APC)·반복횡단면 회귀.

### P5 — Evaluation
- 군집: 안정성(부트스트랩 ARI)·내적(실루엣)·외적(인구·행동 타당도).
- 예측모델: 교차검증·보정(calibration)·SHAP 해석.
- 지수: 강건성표 종합(가중·집계·임계·표준화) + 외부 타당도(디지털뉴스리포트 등 집계 비교).

### P6 — Deployment
- **논문형 PDF**(핵심 산출물) → 그 다음 **Next.js 웹 자가진단 데모**(최종). 웹은 검증 끝난 모델만 탑재.

## 4. 7개년 하모나이즈(Crosswalk) 전략 — 최대 난제

추세·코호트의 성패가 여기 달림. 변수명 직결 불가가 확인됐으므로:

1. **앵커 = 2025**(변수 명세 확정됨). 2025 핵심지표를 기준으로 과거연도 대응 문항을 역추적.
2. **매핑 우선순위**: ① 매체별 이용여부·이용일수(다양성) ② 뉴스/언론 신뢰(전반) ③ 인구(성·연령·지역·학력) — 이 3군이 7개년 모두 존재 가능성 높음(개념적 연속성).
3. **검증**: 매핑된 지표의 연도별 분포가 보고서 통계표와 일치하는지 대조(재검증 게이트).
4. **한계 명시**: 매핑 불가 문항은 추세 제외, 2012~2018 구체계는 범위 외(현 미보유).
5. **리스크**: 문항 워딩 변화로 측정 비동등(measurement non-invariance) 가능 → 추세는 "방향성" 위주 해석, 절대수준 비교는 신중.

## 5. 모델링 기법 위계 — 데이터 규모·성격 적합성

> 사용자 요청: 시계열·ML·DL까지 야심 확장. **단, 전문가적 정직성** — 정형 설문 N≈6천에 무리한 기법은 과적합·실현가능성 감점. 아래는 **"적합성 등급"과 정당화**를 함께 명시한 위계.

| 계층 | 기법 | 적용처 | 데이터 적합성 | 우선순위 |
|------|------|--------|:----:|:--:|
| **기초통계** | 가중 기술통계·교차표·상관 | 전 단계 | ★★★ | P2~ |
| **차원축소·잠재구조** | EFA/CFA·PCA | 신뢰 22문항 잠재차원·구성타당도 | ★★★ | P4-1 |
| **비지도 군집** | K-means(완료)·계층적·GMM | 페르소나 정교화·안정성 비교 | ★★★ | P4-2 |
| **지도 분류** | 로지스틱·결정트리·**GBM(XGBoost/LightGBM)** | 페르소나/건강군 예측 → 웹 자가진단 문항 축소·SHAP 해석 | ★★★ (tabular SOTA) | P4-3 |
| **회귀·인과** | 다중회귀·매개/조절·(준)실험 설계 | 건강지수 결정요인(인구·매체·정치성향) | ★★★ | P4-4 |
| **연관·패턴** | 연관규칙(Apriori/FP-growth) | 매체 이용 조합·동시발생 | ★★☆ | P4-5 |
| **시계열·종단** | 추세검정(Mann-Kendall)·반복횡단면 회귀·**연령-기간-코호트(APC)** | 7개년 신뢰·다양성 추세, 세대효과 분해 | ★★☆ (7시점=짧음 → 전통기법 적합, ARIMA/딥시계열 부적합) | P4-6 |
| **머신러닝 해석** | SHAP·부분의존도·순열중요도 | 모델 설명가능성(대회 시사점) | ★★★ | P5 |
| **딥러닝(조건부)** | tabular DL·임베딩·트랜스포머 | (a) 외부 **비정형 텍스트** 결합 시 임베딩 (b) tabular DL은 GBM 대비 벤치마크로만 | ★☆☆ (정형 6천엔 일반적으로 GBM 우위. **무리한 DL은 과적합·실현가능성 리스크**) | 조건부 |

### 5.1 딥러닝 포지셔닝 (정직한 설계)
- 정형 설문 데이터에서 DL이 GBM을 이기는 경우는 드묾(학계 합의). 따라서 **DL은 "차별화를 위한 무리수"가 아니라 정당화 가능한 트랙**으로만 배치:
  - **트랙 A(권장)**: 비정형 외부 텍스트(뉴스 댓글·SNS·자유응답 — KPF .sav엔 자유응답 비공개이므로 **외부 수집** 필요) → 트랜스포머 임베딩으로 신뢰·회피 정서 보강. [[project-media-stats-unstructured]]의 선택트랙과 연결.
  - **트랙 B(벤치마크)**: tabular DL(TabNet/FT-Transformer) vs GBM 성능 비교를 **방법론 부록**으로 제시 → "DL을 검토했으나 GBM이 우위"라는 분석 성숙도 증거.
- → DL은 P4 후반·조건부. **메인 라인은 GBM + 해석가능 ML**.
