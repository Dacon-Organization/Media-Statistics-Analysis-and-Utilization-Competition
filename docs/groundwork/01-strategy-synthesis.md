## 2. 앵글 결정: A + B 결합

### 검토한 3안
- 🅐 **뉴스 건강 지수 설계 + 코호트** — GAP 직격, 산출물 명확. *(채택)*
- 🅑 **페르소나 세분화 → 맞춤 타겟팅** — 정밀 타겟팅. *(보조 채택)*
- 🅒 영상 플랫폼發 신뢰 인과분석(SEM) — 핫하나 가짜뉴스 레드오션 인접 → **리스크로 보조/심화 분석에만 부분 활용.**

### 결정: **A(척추) + B(보조)**
- A로 "국민 뉴스 건강 지수"를 설계·산출 → 리서치가 명시한 **「표준 측정도구 부재」 정책 GAP**을 정면으로 해소.
- B로 지수를 페르소나로 세분 → 집단별 맞춤 처방으로 **시사점 구체성** 확보.
- C의 인과 관점은 지수 구성요소 검증 단계에서 **부분 차용**(전면에 내세우지 않음).

## 3. 차별화 방어 (규정상 무효 회피)

| 위험 영역 | 이미 존재하는 것 | 우리의 회피·차별 |
|-----------|------------------|------------------|
| 가짜뉴스 대응 | 문체부·방미통위·과기부·KPF센터 | **다루지 않음** (정부 포화) |
| 뉴스 회피 분석 | KPF 보고서·세미나(2024) | 회피도는 지수의 **한 구성요소로만** 사용, 단독 주제화 X |
| 개인화 뉴스앱 | 로제우스·부산일보·아티팩트 | **추천이 아닌 "진단/측정"** 프레임 — 시장 공백 |
| 미디어교육 | KPF미카·운영학교 등 | 교육 자체가 아니라 **"수준 측정 지표"** 제공 (보완재) |

> 핵심: 우리는 **새 서비스를 만든다기보다 "재지 못하던 것을 재는 자(尺)"를 데이터로 만든다.** → 기존 사업과 경합이 아닌 **보완**이라 무효 리스크 낮음.

## 4. 산출물 (Tier 2)

```
논문형 PDF 보고서 (핵심, 15장)
 ├─ 배경·문제정의 (측정도구 부재 GAP)
 ├─ 데이터·방법론 (원시자료 + 코호트/군집)
 ├─ 분석결과 (지수 설계·추세·페르소나)
 ├─ 제안 (B2G 정책 지수 / B2C 진단 서비스)
 └─ 기대효과·실현가능성 (데모 스크린샷 첨부)
+ ZIP: 분석 노트북/코드, 데이터 명세, 설계문서(API/ERD), 웹 데모 소스
+ 웹 데모: Next.js "뉴스 건강검진" (Vercel 배포, 진단 플로우 1개 집중)
```

### 웹 범위 원칙
- **히어로 기능 1개**(진단 플로우)만 실제 구현. 나머지(계정·연도추적·KPF 관리자)는 **설계 문서로 비전 제시**.
- 진단 점수는 분석에서 추출한 **계수/기준값을 브라우저에서 계산** → DB·백엔드 불필요 → 안정 배포.
- 따라서 **API/ERD 명세는 "풀 프로덕션 확장 시" 가정 설계**로 작성.
